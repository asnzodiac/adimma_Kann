Features

Everything packed into one production-ready bot

🎤

Voice-First Design

Send voice messages → auto-transcribed via Groq Whisper → LLM reply → voice response via edge-tts. Full audio pipeline.

Whisper STT edge-tts
🌐

Trilingual Support

Auto-detects English, Malayalam (Unicode), and Manglish (Romanised Malayalam). Replies in the same language. Malayalam TTS uses ml-IN-SobhanaNeural.

🇬🇧 English 🇮🇳 Malayalam 😄 Manglish
👁

Vision / Image Analysis

Send photos → described by Llama 4 Scout Vision → contextual reply. Supports captions + questions about images.

Llama 4 Scout
📄

PDF / Document Reading

Send PDFs and text files. Extracts content with PyPDF2, summarises, and answers questions about the content.

PyPDF2
😴

Sleep / Wake System

Say bye / sleep / standby / good night → bot goes silent. Say hi / hello / wake up / adimma → it comes back. Per-chat state.

🌤

Live Weather

Integrates OpenWeatherMap. Ask "weather in Kochi" naturally — fetches real-time temp, humidity, conditions. Free API tier.

OpenWeatherMap
🔄

API Key Rotation

Up to 4 Groq API keys (GROQ_API_KEY → KEY3) rotated round-robin. Auto-retry on failure. Never hits rate limits.

💾

TTS Caching

All TTS audio cached using MD5(voice + text) as filename. Repeated phrases served instantly. Auto-creates tts_cache/.

📖

Conversation Memory

Remembers last 20 messages per chat. Context-aware replies. Use /clear to wipe history. Each chat has isolated state.

🔔

Owner Notifications

When a new private user starts the bot, the owner (ID: 733340342) gets notified with name, username, and user ID.

🛡

Access Control

Whitelist users via TELEGRAM_ALLOWED_USERS and groups via TELEGRAM_ALLOWED_GROUPS env vars.

📝

Editable Personality

Full character prompt lives in character.txt — edit without touching code. Supports any persona you dream up.

Deployment Guide

From zero to running bot in under 10 minutes

1

Push to GitHub

Create a new GitHub repository and push all 5 files.

# In your project folder:
git init
git add bot.py character.txt requirements.txt render.yaml .gitignore
git commit -m "Initial commit: Adimma Kann bot"
git remote add origin https://github.com/YOUR_USERNAME/adimma-kann.git
git push -u origin main
2

Connect to Render.com

Go to render.com → New → Web Service → Connect GitHub repo → Render auto-detects render.yaml.

3

Set Environment Variables

In Render Dashboard → Environment tab, add these secrets:

TELEGRAM_BOT_TOKEN=your_token
GROQ_API_KEY=gsk_xxx
GROQ_API_KEY1=gsk_yyy (optional)
GROQ_API_KEY2=gsk_zzz (optional)
OPENWEATHER_API_KEY=owm_xxx
OWNER_ID=733340342
4

Deploy & Verify

Click Deploy. Render builds & starts automatically. The bot registers its webhook on startup.

✅ Visit https://your-app.onrender.com/health — should return {"status": "healthy"}

5

Test the Bot!

Open Telegram → find your bot → send /start → Adimma Kann responds! 🎉

# 1. Clone / create project folder
mkdir adimma-kann && cd adimma-kann

# 2. Create virtual environment
python3 -m venv venv && source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Install ffmpeg (required for audio processing)
# Ubuntu/Debian:
sudo apt install ffmpeg
# macOS:
brew install ffmpeg

# 5. Create .env file
cat > .env << EOF
TELEGRAM_BOT_TOKEN=your_token_here
GROQ_API_KEY=gsk_your_key_here
OPENWEATHER_API_KEY=your_owm_key
RENDER_EXTERNAL_URL=https://your-ngrok-url.ngrok.io
EOF

# 6. Expose local port (for webhook)
ngrok http 10000
# Copy the https URL to RENDER_EXTERNAL_URL in .env

# 7. Run the bot
python3 bot.py

Language Detection & TTS Tips

How to improve Malayalam + Manglish experience

🔍 How Language Detection Works

1
Unicode Script Check
If any character is in Malayalam Unicode range (U+0D00–U+0D7F) → classified as Malayalam
2
Manglish Keyword Match
60+ Manglish keywords (njan, enthaaa, alle, pwoli, machaan...). If ≥10% of words match → Manglish
3
langdetect Fallback
If 'ml' detected → Malayalam. Indian languages (ta, te, kn, hi) + Manglish hints → Manglish. Default → English.

💡 Pro Improvement Tips

▸
Expand Manglish keywords: Add more regional words from your area to MANGLISH_KEYWORDS set in bot.py for better detection accuracy.
▸
Add "ml" STT hint by default: Change hint_language default to "malayalam" in transcribe_audio() if most voice messages are Malayalam.
▸
Try ml-IN-MidhunNeural: Alternative Malayalam TTS voice (male). Change TTS_VOICE_MALAYALAM constant in bot.py.
▸
Whisper prompt: Add "Malayalam language, Kerala accent" as prompt parameter to transcribe_audio() for better STT accuracy.
▸
Use whisper-large-v3 (not turbo): Already set. It handles Malayalam significantly better than the turbo model.
